Content Type

Choose the content type you would like to create. You can either choose from the list of core types or one of the templates.

HTML Content

HTML Content will deliver a mobile web page feeling content to mobile devices. The content can be directly rendered by mobile webkit component.

TheCMS provids a WYSIWYG web page editor to help with content construction.

With the help of light weight bridge component, it is able to add Javascript code in the html content to perform native behaviour.[Checkout API]

HTML type content is mainly used for general information or product introduction which needs to push onto mobile devices regularly.

JSON Content

JSON data will deliver a light weight key-value pair data to mobile devices. The data is serialised and can be directly used by any mobile application. For more details about JSON, please [Checkout Here]

The use case for JSON type content is quite common, like the content in dropdown list, the list of files to be downloaded to client, small data storage etc.

Create Now

Table Content

Table data will also deliver a light weight key-value pair data to mobile devices in JSON format. The difference is Table data structure is 'flat' which means it is not nested. Further, it is much simpler to maintain and create compared to JSON type data. Table content use case is a subset of JSON type content like menu list, item list etc.

Create Now

XML Content

XML data has the same functionalities as JSON data but in different format. It is able parsed by mobile application directly as well.

The use case for XML type content is similar to that of JSON type content. It is mainly used for delivering light-weight data.

Create Now

App Form Content

Form data is a fast and simple way to deliver form based content to mobile devices. The output could be either HTML or JSON so that it can be rendered by mobile webkit component or native components.

Form type content is mainly used for form based mobile application so that the form can be designed in CMS and form result can be submitted back to cloud.
